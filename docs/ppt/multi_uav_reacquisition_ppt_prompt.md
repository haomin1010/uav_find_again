## 1. 项目背景

多无人机协同跟随目标时，单架无人机容易因为遮挡、视角变化、目标快速运动、检测器漏检等原因在视觉上跟丢目标。

本项目关注的问题是：当某架无人机失去本机视觉目标时，能否利用集群中其他无人机仍然有效的视觉观测信息，辅助其恢复目标搜索与跟随。

核心目标：

- 构建多无人机协同视觉跟踪框架。
- 支持单机视觉丢失检测。
- 支持集群内目标观测共享。
- 支持丢失无人机基于协同信息进行目标再捕获。
- 建立可演示、可复现、可扩展的 Gazebo 仿真环境。

## 3. 系统总体设计

系统采用分层设计：

感知层：

- 每架无人机从本机视觉获得目标观测。
- 观测可以是 bbox、方位角、距离估计、世界坐标估计、置信度。

协同层：

- 各无人机广播目标观测结果。
- 集群融合多个观测，生成目标全局估计。
- 对过期、低置信度观测进行过滤。

控制层：

- 正常状态下本机视觉闭环跟随。
- 丢失状态下根据集群目标估计进入搜索/再捕获。
- 恢复本机视觉后重新切回本机跟随。

仿真层：

- 使用 Gazebo 构建 3D 场景、无人机模型、目标模型和传感器环境。
- 使用 ROS2 连接仿真、算法和数据记录。

## 5. 最小算法的状态机设计

每架无人机维护一个本地跟踪状态：

- INIT：初始化。
- TRACKING：本机视觉正常跟踪目标。
- LOST：本机连续若干帧未检测到目标。
- REACQUIRING：使用集群目标估计进行再捕获。
- RECOVERED：本机重新检测到目标。
- FAILED：超过恢复时间仍未恢复。

关键转移逻辑：

- 本机检测到目标：进入 TRACKING。
- 连续多帧未检测到目标：进入 LOST。
- LOST 状态下收到其他无人机可用目标估计：进入 REACQUIRING。
- REACQUIRING 状态下本机重新检测到目标：进入 RECOVERED。
- RECOVERED 稳定若干帧后：回到 TRACKING。

## 6. 当前最小算法的数据流

当前 demo 的数据流：

```text
目标轨迹
  -> 无人机位置与朝向
  -> mock 视觉观测
  -> 本机跟踪状态判断
  -> 集群目标估计融合
  -> 丢失无人机再捕获控制
  -> 轨迹、事件和视频记录
```

当前 mock 视觉观测包含：

- detected：是否检测到目标。
- confidence：观测置信度。
- loss_reason：丢失原因。
- target_position_estimate：目标位置估计。
- bearing：目标相对方位角。
- range_estimate：目标距离估计。

## 8. 已完成的 ROS2 代码结构

当前已搭建 ROS2 第一版代码结构：

```text

  multi_uav_sim_msgs/
  multi_uav_sim_nodes/

```

消息定义：

- `VisionObservation`
- `TrackerState`
- `SwarmTargetEstimate`
- `SimEvent`
- `UavPose2D`

节点定义：

- `target_manager`：发布目标真实位置。
- `uav_control`：发布无人机位置和朝向。
- `vision_mock`：生成 mock 视觉观测。
- `coop_tracker`：处理状态机、集群融合和再捕获。
- `sim_recorder`：记录事件、轨迹和指标。

启动方式：

```bash
ros2 launch multi_uav_sim_bringup mock_demo.launch.py
```

## 11. Gazebo 与当前 ROS2 代码的关系

当前 ROS2 代码已经把算法和仿真环境解耦。

后续接入 Gazebo 时，主要替换三个部分：

```text
当前 target_manager
  -> Gazebo 中的目标模型与轨迹控制

当前 uav_control
  -> Gazebo 无人机模型控制，后续可接 PX4

当前 vision_mock
  -> Gazebo 相机、分割、bbox 或真实视觉检测器
```

保留部分：

- `multi_uav_sim_msgs`
- `coop_tracker`
- `sim_recorder`
- 状态机逻辑。
- 协同融合逻辑。
- 事件与指标记录。

这样的设计可以避免算法被某个仿真器或视觉后端强绑定。

## 13. 后续算法增强方向

当前算法是最小基线，后续可以增强：

融合算法：

- 加权平均。
- EKF/UKF。
- 粒子滤波。
- 多假设跟踪。

恢复策略：

- 直接追踪集群估计位置。
- 环绕搜索。
- 预测目标运动方向并前方拦截。
- 根据多机位置重新分配视角。

鲁棒性建模：

- 通信延迟。
- 通信丢包。
- 观测噪声。
- 长时间遮挡。
- 多目标干扰。
